# 就业补充课-JVM

## 一、JVM是什么

JVM是Java Virtual Machine的缩写，JDK (Java Development Kit)= JRE(Java Runtime Environment)  + JVM。

![image-20250214093332239|800](https://woniumd.oss-cn-hangzhou.aliyuncs.com/security/mayuhang/202502140933389.png)

为什么要学习Java虚拟机？

> 所有的高级程序员，以及架构师，对于JVM，都很了解。同时，它也是大家提供薪水必备的知识。还有懂了JVM，就可以更加理解Java语言。
>
> 大家达到什么程度呢？用大脑当作JVM，边看边脑袋运行代码。	

JVM对于Java的作用：跨平台，内存管理，性能优化，安全性，垃圾回收，字节码执行等	

![image-20250214093636319|750](https://woniumd.oss-cn-hangzhou.aliyuncs.com/security/mayuhang/202502140936448.png)

JVM很重要，知道了虚拟机，你就知道了代码怎么运行，

- [ ] 什么方法，什么是变量，什么是类，什么静态类，什么静态方法，什么静态代码块，他们的执行顺序是什么？
- [ ] 常量池在哪里，以及方法区（元空间）用来干嘛？
- [ ] 为什么大家很喜欢用hashcode的值在哪里？synchronized锁，锁的是什么？0，10,11 锁升级又是什么鬼？
- [ ] new一个对象会发生什么？对象里面有什么？
- [ ] 以及数组的长度属性在哪里？
- [ ] 垃圾回收为什么不要太频繁?
- [ ] 类加载机制，双亲委派?
- [ ] 反射到底反了个什么？
- [ ] 字节码class文件怎么解读？
- [ ] JVM的运行时数据区的所有内容？堆、栈（栈帧，本地方法栈，局部变量表）、新生代，老年代，永久代（元空间），TLAB, S0, S1, 栈帧（局部变量表，操作数组，动态链接，方法返回）
- [ ] 更多还需要了解计算机底层原理，知晓CPU的一个结构（JUC并发的课程中讲）。
- [ ] new String("a")+new String("b") ；一共创建了多少个对象。







## 二、运行时数据区

### 1. 栈区

Stack区域（stack segment）— 由编译器自动分配释放,存放**函数的参数值**，**局部变量**的值等，具体方法执行结束之后，系统自动释放JVM内存资源
