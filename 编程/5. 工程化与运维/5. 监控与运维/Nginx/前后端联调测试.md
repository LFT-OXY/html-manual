
# 1.反向代理

反向代理是一种网络架构技术，通过反向代理服务器为后端服务器做代理（安全、灵活、负载均衡）

1. location：用于定义匹配路径匹配的规则。
2. ^~ /api/：表示精确匹配，即只匹配以/api/开头的路径。
3. rewrite：该指令用于重写匹配到的路径。
4. proxy_pass：该指令用于代理转发，它将匹配到的请求转发给位于后端的指令服务器。
