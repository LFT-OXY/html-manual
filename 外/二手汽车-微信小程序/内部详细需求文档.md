# 车辆第三方检测 + 微信小程序报价平台（内部需求文档）

下面是一份纯业务视角的项目需求 / 流程文档草案，可用于立项说明或产品需求文档。

------

## 一、项目背景与目标

- 公司现有车辆在出售前，车况信息主要依赖内部检查或简单描述，透明度有限，经销商和内部员工报价不够精准。
- 引入第三方车辆检测机构，对车辆做标准化检测并出具检测报告，通过微信小程序展示车辆详情（含图片、第三方检测报告），让车商或内部员工在线报价。
- 目标：

- 提升车辆检测标准化程度与可信度；
- 提升车辆报价效率与准确性；
- 统一车辆在售管理流程，沉淀数据。

------

## 二、系统角色定义

1. **车辆管理员（运营人员）**

- 负责车辆录入、检测进度跟踪、报告上传、车辆在售状态控制、选择/确认报价等。

1. **经销商用户（外部车商）**

- 查看车辆详情和第三方检测报告，提交报价，跟踪自己的报价状态。

1. **内部员工（内部购车或推荐）**

- 与经销商类似，查看车辆信息并报价，可能享有特殊标识或权限。

1. **系统管理员**

- 管理账号与权限、基础配置（检测机构信息、车辆品牌/车型库等）、查看统计报表。

------

## 三、整体业务流程（概览）

1. **车辆录入**

- 车辆管理员录入车辆基本信息（车牌号、VIN、品牌车型、上牌时间、里程等）。

1. **第三方检测申请**

- 选择合作检测机构，提交车辆检测申请（可记录预约时间、地点等信息）。

1. **车辆检测与报告生成**

- 第三方机构完成检测，生成车辆检测报告（PDF/图片/网页链接等形式）。

1. **检测报告上传/关联**

- 车辆管理员将第三方检测报告上传或录入报告链接，并在系统中与该车辆绑定。

1. **车辆信息发布**

- 管理员完善车辆照片、价格参考信息，设置可报价时间和规则，将车辆发布到微信小程序。

1. **用户查看与报价**

- 经销商/内部员工通过小程序查看车辆详情与检测报告，按规则提交报价。

1. **报价管理与选定**

- 管理员查看各车辆的报价列表，对比价格与用户类型，选定一个或多个报价（中标/待谈判）。

1. **成交 / 失败关闭**

- 选定报价后，线下或线上完成后续合同与过户流程，在系统中将车辆状态更新为“已成交”或“售卖失败/下架”。

------

## 四、车辆状态与流转说明

建议的车辆状态：

1. `待检测`：已录入车辆信息，尚未提交第三方检测。
2. `检测中`：已向检测机构申请，等待完成报告。
3. `已出报告-待发布`：检测报告已上传/关联，尚未对外发布。
4. `在售-可报价`：车辆已发布到小程序，允许用户查看和报价。
5. `报价中-待选择`：已收到至少一个报价，等待管理员选择。
6. `已选定报价-待成交`：管理员已选定某一报价，等待实际成交处理。
7. `已成交`：车辆已成功售出，流程结束。
8. `已下架/已作废`：车辆因各种原因不再出售（例如内部自用、退回、报废等）。

------

## 五、主要功能模块需求

### 1. 车辆管理

- **车辆录入**

- 必填字段：车牌号、车架号（VIN）、品牌、车型、车身颜色、上牌日期、公里数、排量/动力类型（燃油/混动/纯电）、变速箱类型、排放标准、车辆来源（公司自有/回购/其他）。
- 可选字段：购置日期、过户次数、年检到期时间、保险到期时间、当前所在地等。

- **车辆详情维护**

- 支持管理员编辑车辆信息（除关键字段如 VIN 可限制编辑）。
- 支持记录备注信息（维修记录、已知问题等）。

- **车辆图片管理**

- 支持上传多张照片（外观多角度、内饰、发动机舱、底盘、仪表里程、事故痕迹等）。
- 可标记“封面图”，可调整展示顺序。

- **状态管理**

- 管理员可将车辆状态从一个阶段切换到下一阶段（例如：待检测 → 检测中 → 已出报告-待发布 → 在售-可报价）。
- 记录每次状态变更的时间与操作者。

### 2. 第三方检测报告管理

- **检测机构信息维护**

- 管理合作检测机构名称、联系人、联系方式、检测地址等。

- **检测申请记录**

- 针对每辆车记录检测申请时间、预计完成时间、实际完成时间。

- **检测报告上传/关联**

- 支持上传报告文件（如 PDF、图片）或录入报告链接。
- 可录入报告关键数据摘要（例如：结构件是否变形、是否有重大事故、漆面修复位置、故障码、检测评分等）。
- 报告版本管理：如同一辆车多次检测，可保留历史记录，只展示最新版本给用户。

- **报告展示**

- 在车辆详情页中，以清晰结构展示检测结果摘要，并提供“查看完整报告”的入口。

### 3. 报价管理

- **报价规则配置**

- 是否限制报价时间段（报价开始/结束时间）。
- 是否限制每个用户对同一车辆的报价次数（如只能报一次或允许修改）。
- 是否需设置最低/最高可报价范围（保护价）。

- **报价提交**

- 报价页面展示：车辆核心信息、检测报告摘要、当前报价情况（视规则可展示或隐藏其他报价信息）。
- 用户填写内容：

- 报价金额；
- 有效期/锁价期限（可选）；
- 备注（例如：需要整备、需要更换部件等条件）。

- **报价列表与排序**

- 管理员查看每辆车所有报价，支持按报价金额、时间、用户类型（经销商/员工）排序。
- 在车辆详情后台页面展示“最高价、最低价、平均报价”等统计信息。

- **选定报价与状态更新**

- 管理员可将某个报价标记为“中选/待成交”。
- 被选中的报价对应车辆状态变更为“已选定报价-待成交”。
- 可记录最终成交金额（如与报价有调整），更新为“已成交”。

- **报价结果通知**

- 对于未中选用户，系统可发送“报价未被采纳”的通知。
- 对于中选用户，发送“报价已被选中，请联系/等待联系”的通知。

### 4. 用户与权限管理

- **用户类型**

- 经销商用户：具备查看在售车辆和提交报价权限。
- 内部员工：同样可以查看和报价，可配置是否有特殊价格权限、是否单独统计。
- 管理员：车辆管理、报告管理、报价管理、用户管理。

- **用户注册与审核**

- 经销商注册：提交公司名称、联系人、电话、营业执照照片等，管理员后台审核后开通。
- 内部员工：可通过员工工号/手机号由管理员导入或关联。

- **权限控制**

- 不同角色看到的页面和功能不同，例如经销商看不到内部管理数据，员工可能有内部专属车辆等。
- 支持禁用/启用某个用户账号。

### 5. 微信小程序前端功能（用户端）

- **首页 / 车辆列表**

- 车辆列表展示：缩略图、品牌车型、年款、里程、所在地、当前状态（在售）、是否新上架标识等。
- 筛选条件：品牌、价格区间、上牌年份、里程范围、所在地、检测结果（无重大事故/有钣金喷漆等）。

- **车辆详情页**

- 车辆基础信息展示。
- 照片浏览（轮播或九宫格）。
- 检测报告摘要 + “查看完整报告”入口。
- 历史报价概况（视配置是否向所有用户展示，如仅显示“已有 X 条报价”）。
- 报价入口/按钮（满足可报价时间和用户权限时可用）。

- **报价页**

- 展示确认信息（车辆信息、注意事项、规则）。
- 填写报价金额、备注。
- 提示：报价提交后是否可修改，是否需要联系客服等。

- **我的报价**

- 用户可查看自己所有报价记录：报价车辆、报价金额、时间、当前状态（待选定/已中选/未中选/已成交等）。

- **通知中心**

- 报价结果通知、车辆状态变更相关通知（例如：关注车辆已成交/已下架）。

### 6. 管理端功能（可在小程序内的“管理入口”，或独立后台）

- **车辆管理列表**

- 支持按状态筛选车辆。
- 支持批量操作（例如批量发布或批量下架）。

- **报价管理列表**

- 针对单车查看所有报价详情。
- 全局查看最近报价、最高报价记录等。

- **报表与统计（初期可简单）**

- 一段时间内车辆检测数量、在售数量、成交数量。
- 不同渠道（经销商/员工）成交占比。
- 平均销售周期（从在售到成交的天数）。

## 六、关键页面与操作流程说明

1. **管理员新增车辆流程**

- 进入“车辆管理” → 点击“新增车辆” → 填写基本信息 → 保存为“待检测”状态。

1. **管理员发起检测流程**

- 在车辆详情中选择检测机构 → 填写预约时间 → 状态变更为“检测中” → 等待第三方检测。

1. **管理员上传检测报告流程**

- 检测完成后获取报告 → 在车辆详情中上传报告文件或粘贴报告链接 → 填写报告摘要 → 状态变更为“已出报告-待发布”。

1. **车辆发布在售流程**

- 管理员补充/确认车辆照片、配置报价规则（时间、限制等） → 一键发布 → 状态改为“在售-可报价”，同时在小程序首页展示。

1. **用户查看并报价流程**

- 用户打开小程序 → 浏览车辆列表 → 进入某车详情 → 查看检测报告 → 点击“我要报价” → 填写报价 → 提交 → 用户在“我的报价”中可查看记录。

1. **管理员选择报价与成交流程**

- 管理员在后台打开车辆报价列表 → 对比各报价 → 选择合适报价标记为“中选” → 联系用户完成线下成交 → 在系统中输入最终成交金额并更新车辆状态为“已成交”。

------

## 七、非功能性需求（简要）

- **使用便捷性**

- 界面简洁、步骤明确，车辆管理员在手机端也能完成主要操作。

- **性能与稳定性**

- 支持并发访问常见规模（如几十到几百经销商同时浏览与报价）。
- 图片和报告加载速度要可接受，可考虑压缩与分页展示（在设计与实现层面考虑）。

- **安全与隐私**

- 报告与车辆详细信息仅对已登录用户开放。
- 经销商和员工的报价金额对其他用户是否可见，需要在规则中明确（默认不公开具体金额，仅中后台可见）。

- **日志与审计**

- 记录关键操作日志：车辆状态变更、报告上传、报价新增/修改、报价选定等，以便后续追踪。
