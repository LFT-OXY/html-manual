# 一、项目概述

- **项目名称**：车辆第三方检测 + 微信小程序报价平台  
- **核心思路**：
  在车辆对外/内部出售前，统一送至第三方检测机构出具专业检测报告，并在微信小程序中展示车辆照片和检测结果，方便车商及内部员工在线报价，实现“车况透明 + 报价高效”。

------

## 二、项目目标与价值

### 1. 对公司

- 提升车辆处置效率：从“线下沟通零散报价”升级为“线上集中展示与集中报价”。
- 提高车辆出售价：凭借第三方检测报告，提升车况透明度，吸引更多车商参与竞价。
- 规范流程与数据沉淀：统一车辆录入、检测、发布、报价、成交等流程，形成可追溯数据。

### 2. 对合作车商 / 内部员工

- 获取信息更完整：可在线查看车辆详细照片和独立第三方检测报告。
- 报价更方便：不必到现场即可初步评估车况，直接在小程序报价。
- 合作更透明：报价结果、车辆状态有明确记录，减少信息不对称。

### 3. 对第三方检测机构

- 稳定检测业务来源：公司车辆集中送检，形成长期合作。
- 品牌曝光：检测报告通过小程序展示给车商、员工，提高机构影响力。

------

## 三、使用场景示例

1. **公司计划批量出售一批车辆**
   车辆统一送至合作检测机构 → 出具检测报告 → 小程序发布 → 车商和员工集中报价 → 选定最佳报价 → 线下完成交易。
2. **内部员工想购买公司车辆**
   员工通过小程序查看车辆详情与检测报告 → 与车商一起公平竞价，或享受专属员工购车通道。
3. **车商日常收车**
   车商关注小程序新上架车辆 → 在线筛选符合自己需求的车 → 快速报价，提升业务效率。

## 四、整体业务流程（简化版）

1. **车辆录入**
   由车辆管理员录入车辆基础信息（品牌车型、车牌号、里程、上牌时间等）。
2. **第三方检测**
   管理员在系统中为车辆发起检测申请 → 车辆送检 → 检测机构出具标准化报告。
3. **报告上传与审核**
   管理员将检测报告（文件或链接）上传并关联到对应车辆 → 简要录入关键结论信息。
4. **车辆发布**
   补充车辆照片、基本介绍 → 设置是否开放给车商、内部员工报价 → 在微信小程序中发布。
5. **车商 / 员工报价**
   用户在小程序中浏览车辆 → 查看检测报告和照片 → 在规定时间内提交报价，并可查看自己的报价记录。
6. **报价评估与成交**
   管理员对比各方报价和用户类型 → 选定合适报价 → 发起线下合同和过户流程 → 在系统中记录成交情况。

## 五、关键功能概览

### 1. 管理端（公司内部使用）

可在小程序管理入口或独立后台实现，核心包括：

- **车辆管理**

- 新增/编辑车辆信息；
- 上传车辆照片；
- 设置车辆状态（待检测、在售、已成交等）。

- **检测管理**

- 选择第三方检测机构；
- 记录检测申请信息；
- 上传并维护检测报告。

- **报价管理**

- 查看每辆车的所有报价；
- 对比报价金额、时间、报价方类型；
- 选择中标报价；
- 记录成交价格。

- **用户与权限管理**

- 车商账号审核与维护；
- 内部员工账号管理；
- 角色权限区分（仅查看 / 可报价 / 可管理）。

- **数据统计（初期可简化）**

- 车辆在售数量、成交数量；
- 车商与内部员工成交占比；
- 平均销售周期等。

### 2. 用户端（微信小程序）

- **车辆列表**

- 展示在售车辆的核心信息：品牌、车型、里程、上牌年份、照片缩略图等；
- 支持按品牌、价格、里程等条件筛选。

- **车辆详情**

- 展示车辆基础信息和多角度照片；
- 展示第三方检测报告的概要结论和完整报告入口；
- 展示车辆当前状态（在售/已成交）。

- **报价功能**

- 用户在线填写报价金额和备注；
- 系统提示报价规则（如是否允许修改、是否有限定时间等）。

- **我的报价**

- 用户查看历史报价记录以及每笔报价的处理状态（待选定/未中选/已中选/已成交等）。

- **消息通知**

- 报价提交成功提醒；
- 报价结果通知；
- 关注车辆状态变更提醒等。

## 六、合作方参与方式（建议）

### 1. 第三方检测机构

- **合作模式**

- 按单收费或批量包年/包量；
- 约定检测内容和标准（事故检测、结构件检测、漆面检测、故障码检测等）。

- **权责划分**

- 机构负责按约定标准出具客观报告；
- 公司将报告原样展示于小程序；
- 双方约定免责条款和争议处理机制。

### 2. 车商合作方

- **接入方式**

- 通过微信小程序注册或由公司审核开通；
- 实名登记公司与联系人信息。

- **合作规则**

- 明确报价规则（是否允许多次报价、是否公开他人报价、是否有保证金机制等）；
- 明确交易后续流程（中选后联系方式、合同签订方式、付款方式等）。

### 3. 内部员工

- **参与方式**

- 关联员工信息，开放指定车辆或统一开放报价；
- 可单独设置员工专属政策（如优先购买权、内部价格策略等）。

## 七、实施节奏与里程碑（建议）

可按三个阶段推进，便于控制风险和投入：

1. **第一阶段：流程跑通（约 X 周）**

- 目标：实现“车辆录入 → 检测 → 报告上传 → 发布 → 报价 → 成交记录”的完整闭环，界面以功能为主。
- 范围：内部管理员 + 少量合作车商试运行。

1. **第二阶段：优化与扩展（约 X 周）**

- 目标：优化操作体验和图片/报告展示效果；补充数据统计报表。
- 范围：更多车商接入；开放给内部员工使用。

1. **第三阶段：规模推广与精细运营**

- 目标：通过数据分析优化价格策略、销售周期；
- 考虑接入更多检测机构和销售渠道。

------

## 八、风险与控制要点（简要）

- **检测报告的权威性与责任界定**

- 通过合同明确第三方检测机构的责任范围和免责条款；
- 在小程序中说明“报告来源于第三方机构”，公司仅做展示。

- **报价与成交的合规性**

- 在报价页面和规则说明中说明：报价不等于最终成交，最终以合同为准；
- 明确违约与放弃的处理方式。

- **数据与权限安全**

- 报价金额和详细数据仅对公司管理端可见；
- 普通用户仅根据规则看到必要信息，避免敏感信息外泄。
