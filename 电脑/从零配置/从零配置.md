点进软件更新后点立即更新，之后的弹窗同意和不同意都不要点，调出活动监视器，然后快捷键 command+option+esc，强制退出 `系统设置` 的进程，然后就成了

### 2. 屏蔽更新

修改 hosts 文件，屏蔽系统更新检测联网

打开终端，执行命令：

```
sudo vim /etc/hosts
```

按 i 进入编辑模式，hosts 文件添加以下内容：

```plain
# mac software update
127.0.0.1 swscan.apple.com
127.0.0.1 swdist.apple.com
127.0.0.1 swdownload.apple.com
127.0.0.1 swcdn.apple.com
127.0.0.1 updates-http.cdn-apple.com
127.0.0.1 updates.cdn-apple.com
127.0.0.1 xp.apple.com
127.0.0.1 gdmf.apple.com
127.0.0.1 mesu.apple.com
127.0.0.1 appldnld.apple.com
```

以上内容为 mac 获取系统更新的地址，屏蔽掉则无法检测出有新系统更新。编辑完毕后，按 esc，然后按: wq 或者:wq! 强制保存 hosts 文件修改。

1. **保存并退出**：

- 按 `Ctrl + O` → `Enter` 保存。

- 按 `Ctrl + X` 退出。

1. **刷新 DNS 缓存**：

```plain
sudo dscacheutil -flushcachesudo

```

### 3. MacOS 15 Sequoia 安全性关闭的两种方法@打开任何来源办法

#### **第一种方式**

在 macOS 中，安全性功能是为了保护用户的隐私和设备安全而设计的。关闭这些功能可能会降低系统的安全性，因此建议仅在必要时进行操作，并确保了解相关风险。

MacOS 15（假设您指的是 macOS Sequoia 或其他未来版本）的安全性设置可能包括以下内容：

**1、系统完整性保护 (SIP)**

**2、Gatekeeper**

**3、防火墙**

**4、文件保险箱 (FileVault)**

**5、隐私权限管理**

以下是关闭这些功能的具体步骤：

------

##### **一、关闭安全性**

###### **1. 关闭系统完整性保护 (SIP)**

系统完整性保护是 macOS 的核心安全机制，用于防止未经授权的修改系统文件。

###### 步骤：

**1、重启 Mac 并进入恢复模式**

- 

- 重启电脑，在启动时按住 `Command (⌘) + R` 键，直到看到 Apple 标志或旋转的地球图标。

**2、打开终端**

- 

- 在恢复模式下，点击顶部菜单栏的“实用工具”，然后选择“终端”。

**3、禁用 SIP**

- 

- 在终端中输入以下命令并按下回车：

```plain
csrutil disable
```

- 终端会提示已成功禁用 SIP。

**4、重启电脑**

- 

- 输入 `reboot` 或直接点击苹果菜单重启。

**注意** ：关闭 SIP 可能会导致系统不稳定或安全漏洞，请谨慎操作。

------

##### **2. 关闭 Gatekeeper**

Gatekeeper 是 macOS 的一项功能，用于确保仅运行来自 App Store 或受信任开发者的应用程序。

###### 步骤：

1、打开 **终端** 。

2、输入以下命令并按下回车：
开启：
`sudo spctl --master-disable`

关闭：
`sudo spctl --master-enable`

3、输入管理员密码并确认。

4、如何验证是否成功？

**检查 Gatekeeper 状态**：

```
spctl --status
```

- 如果返回 `assessments disabled`，说明已关闭。

- 如果返回 `assessments enabled`，说明仍在启用状态。

完成后，您可以在“系统设置” > “隐私与安全性”中看到“任何来源”选项被启用。

------

##### **3. 关闭防火墙**

MacOS 防火墙可以阻止未经授权的网络连接。

###### 步骤：

1、打开 **系统设置** > **网络** > **防火墙** 。

2、点击左下角的锁图标，输入管理员密码解锁。

3、点击“关闭防火墙”。

------

##### **4. 关闭文件保险箱 (FileVault)**

FileVault 是 macOS 的磁盘加密功能，用于保护您的数据。

###### 步骤：

1、打开 **系统设置** > **隐私与安全性** > **文件保险箱** 。

2、点击“关闭 FileVault”。

3、系统会提示解密磁盘，此过程可能需要一些时间。

------

##### **5. 调整隐私权限**

MacOS 允许用户控制应用程序对麦克风、摄像头、位置等的访问权限。

###### 步骤：

1、打开 **系统设置** > **隐私与安全性** 。

2、在左侧选择具体的权限类别（如“麦克风”、“摄像头”等）。

3、取消勾选不需要授权的应用程序。

在 macOS 中，关闭某些安全功能后，您可以通过以下方法来确认是否成功关闭。以下是针对不同功能的具体检查方法：

在 Windows 10 中，通过注册表删除开机启动项的步骤如下‌：

‌打开注册表编辑器‌：按下 Win + R 键，输入 regedit，然后按回车键，打开注册表编辑器。
‌导航到启动项的位置‌：注册表中的启动项通常位于以下几个位置：
`HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run`：这是用户级别的启动项，删除这些项目通常不会影响系统运行。
`HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Run`：这是系统级别的启动项，通常是第三方软件的驱动程序。删除这些项目时需要谨慎，以免影响系统正常运行。
`HKEY_LOCAL_MACHINE\SOFTWARE\WOW 6432 Node\Microsoft\Windows\CurrentVersion\Run`：这个位置通常包含与操作系统正常运行相关的启动项，删除时需要特别小心。
‌删除启动项‌：在注册表编辑器中，找到上述路径下的相应启动项。右键点击要删除的项目，选择“删除”。删除前建议先备份注册表，以防误操作导致系统问题。
‌注意事项‌：

在修改注册表之前，务必创建备份，以防不小心修改导致系统问题。
删除注册表项时需要谨慎，尤其是系统级别的启动项，错误的操作可能会对系统造成不可逆的损伤。
如果不确定某个启动项的作用，可以先禁用而不是直接删除，观察系统反应再决定是否彻底删除。
通过以上步骤，你可以有效地管理和删除 Windows 10 的开机启动项，从而提高系统启动速度和性能

## 5. Win 11 跳过联网

跳过强制使用 Microsoft 登录的限制：

方法一：使用 OOBE 命令来安装，自从 2022 年 9 月底发布了 Windows 11 22 H 2 版本后，全新安装系统时再也无法使用本地账户登录了。以前的做法是断网就可以跳过强制使用 Microsoft 登录的限制，但新版本会提示要联网才能继续安装。

在没有网络连接的情况下，“下一步”按钮是灰色不可点击的。这时候，按下 Shift + F 10 键打开命令提示符（使用笔记本的朋友如果无法打开命令提示符，可以按 Fn + Shift + F 10）。

出现命令提示符后，输入“oobe\bypassnro”，然后按回车键。当按下回车键后，计算机会重新启动。

重新启动后，将回到设置国家区域的界面，然后继续 Windows 11 的设置过程。这时在网络界面上终于出现了“我没有 Internet 连接”的选项，点击进入。



## 6. 隐藏 Windows 11 主文件夹和图库

Windows 11 文件资源管理器左侧的主文件夹和图库删除或者隐藏不能通过常规界面操作来实现，要隐藏 Windows 11 主文件夹和图库，需要修改系统注册表。

Windows 11 主文件夹（Home）在注册表的位置是

HKEY_CLASSES_ROOT\CLSID{f874310e-b6b7-47dc-bc84-b9e6b38f5903}
Windows 11 图库（Gallery）在注册表的位置是

HKEY_CLASSES_ROOT\CLSID{e88865ea-0e1c-4e20-9aa6-edcd0212c87c}
cut-off



将各自注册表项隐藏主文件夹和图库的 System.IsPinnedToNameSpaceTree 从 1 改成 0 就可以让主文件夹和图库消失。

不过，这些注册表项有权限限制，需要先为用户指派足够的权限才能修改它们的值。操作步骤如下。

以管理员身份运行 regedit.exe
定位到上述注册表项
在注册表项上单击鼠标右键，选择“权限(P)”
在弹出的“安全”窗口中点击“高级(V)”
将“所有者”更改为“Authenticated Users”
为 &#34;Administrators&#34;分配“完全控制”权限
将 System.IsPinnedToNameSpaceTree 改成 0
恢复修改前的权限并将“所有者”改回原始值 NT Service\TrustedInstaller
